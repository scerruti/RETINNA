Root Cause Analysis
-------------------

Class Imbalance in Test Set:
  - Low Severity: 14,805,762 px (83%)
  - Water: 1,405,126 px (8%)
  - Cloud: 996,743 px (6%)
  - Extreme: 161,836 px (0.9%) ← CRITICAL, UNDERREPRESENTED
  - Moderate: 428,606 px (2.4%)
  - High: 27,591 px (0.2%)
  - Unburned: 128 px (0.0%)

Effect on Model:
  The model learned to predict "Low Severity" as the default safe strategy
  because it represents 83% of training data. Weighted cross-entropy loss
  and z-score normalization were insufficient to overcome the imbalance.

  Minority classes (Extreme, High, Moderate, Unburned) are treated as noise.

Recommendations for Improvement
--------------------------------

1. Address Class Imbalance (High Priority):
   - Focal loss (α=0.25, γ=2.0) instead of weighted cross-entropy
   - Oversampling minority classes during training
   - Rebalanced datasets or stratified sampling

2. Improve Extreme Detection:
   - Class weighting: increase weight of Extreme class further
   - Weighted dice loss or Tversky loss designed for imbalance
   - Custom loss function penalizing Extreme misclassification

3. Architecture Considerations:
   - Revert to 4-channel difference model (appears to perform better)
   - Or keep 8-channel but fix loss function/weighting

4. Data Augmentation Tuning:
   - Current augmentation (flip, rotate, zoom/crop) may be hurting performance
   - Consider less aggressive augmentation for imbalanced data
   - Augment minority classes specifically

5. Validation Strategy:
   - Run baseline 4-channel inference for direct comparison
   - Check if problem is architectural or just loss/weighting
