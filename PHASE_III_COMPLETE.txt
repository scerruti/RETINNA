================================================================================
                         ARCHITECTURE & DESIGN
================================================================================

INPUT (Post-Fire Sentinel-2):
  └─ Red Band (B04) + NIR Band (B08) → 2 channels
     ├─ NDVI = (NIR - Red) / (NIR + Red + 1e-8)
     └─ NIR (raw reflectance)

MODEL (Binary U-Net):
  └─ Encoder-Decoder with skip connections
     ├─ Input: [N, 2, H, W]
     ├─ 5 downsampling blocks (with DoubleConv)
     ├─ 5 upsampling blocks (with skip connections)
     └─ Output: [N, H, W] (binary logits)

TRAINING (BCEWithLogitsLoss):
  └─ Adaptive pos_weight: n_no_burn / n_burn
  └─ Data augmentation: Flip, Rotate, Zoom/Crop
  └─ Learning rate scheduling: ReduceLROnPlateau
  └─ Optimizer: Adam (lr=1e-3, weight_decay=1e-5)

EVALUATION (Binary Classification):
  └─ Metrics: Accuracy, Precision, Recall, F1, IoU
  └─ Critical: RECALL = "% of actual burns detected"
  └─ Confusion matrix: TP, FP, TN, FN

================================================================================
                         LOSS FUNCTIONS
================================================================================

Available in phase3_losses.py:

1. BCEWithLogitsLoss (DEFAULT)
   └─ Baseline binary cross-entropy with pos_weight
   └─ Stable, simple, effective for class imbalance

2. FocalLoss
   └─ Focuses on hard-to-classify pixels
   └─ Good for extreme class imbalance
   └─ Parameters: α (weighting), γ (focus, default 2.0)

3. DiceLoss
   └─ Direct F1/Dice coefficient optimization
   └─ More stable for minority classes
   └─ Parameter: smooth (default 1e-6)

4. TverskyLoss
   └─ Tunable false positive vs false negative trade-off
   └─ Generalizes Dice loss
   └─ Parameters: α (FP penalty), β (FN penalty)

5. CombinedLoss
   └─ Weighted sum of multiple losses
   └─ Example: BCE + Dice for robustness

Quick iteration: Edit phase3_config.py, restart kernel, re-train
